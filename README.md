# oh-my-claudeagent

Turn Claude Code into a multi-agent system. Specialist agents handle planning, execution, code review, debugging, and research â€” with persistence modes, parallel execution, and natural language activation.

## Installation

Add the marketplace and install:

```bash
claude plugin marketplace add UtsavBalar1231/oh-my-claudeagent
claude plugin install oh-my-claudeagent@omca
```

Or from inside a Claude Code session:

```
/plugin marketplace add UtsavBalar1231/oh-my-claudeagent
/plugin install oh-my-claudeagent@omca
```

After installing, run `/oh-my-claudeagent:omca-setup` to configure your environment and verify dependencies.

### Team Setup

Add to your project's `.claude/settings.json` so team members get the plugin automatically:

```json
{
  "extraKnownMarketplaces": {
    "omca": {
      "source": {
        "source": "github",
        "repo": "UtsavBalar1231/oh-my-claudeagent"
      }
    }
  },
  "enabledPlugins": {
    "oh-my-claudeagent@omca": true
  }
}
```

### Update

```bash
/plugin marketplace update omca
```

### Uninstall

```bash
/plugin uninstall oh-my-claudeagent@omca
```

## Requirements

- Claude Code CLI
- `jq`
- `uv`
- `python3` 3.10+
- `ast-grep` CLI (`ast-grep` or `sg`)

## Agents

A master orchestrator delegates to specialists based on task type. Every agent has a Greek mythology name.

| Agent | Role | Model |
|-------|------|-------|
| **sisyphus** | Master orchestrator â€” routes work to the right specialist | opus |
| **sisyphus-junior** | Focused single-task executor | sonnet |
| **prometheus** | Strategic planner â€” interviews you, then builds a plan | opus |
| **metis** | Pre-planning analyst â€” finds requirement gaps before you start | opus |
| **momus** | Plan reviewer and critic | opus |
| **oracle** | Architecture advisor and read-only verifier | opus |
| **atlas** | Todo-list orchestrator â€” tracks and drives plan execution | opus |
| **explore** | Codebase search specialist | sonnet |
| **librarian** | External research â€” SDK docs, API references | sonnet |
| **hephaestus** | Build and toolchain fixer | sonnet |
| **multimodal-looker** | Vision, PDF, and image analysis | sonnet |
| **socrates** | Deep research interview using Socratic method | opus |

Override any agent's default model with `Agent(model="haiku")` or `Agent(model="opus")` for cost or quality tuning.

## Skills

Invoke with `/oh-my-claudeagent:<skill>` in any session.

| Skill | What it does |
|-------|-------------|
| `ralph` | Persistence mode â€” keeps working until verified complete |
| `ultrawork` | Maximum parallel execution across independent tasks |
| `refactor` | Refactoring with zero-regression verification |
| `init-deep` | Generate `AGENTS.md` files across your codebase |
| `prometheus-plan` | Strategic planning via interview workflow |
| `metis` | Pre-planning analysis and gap detection |
| `start-work` | Execute a plan generated by prometheus |
| `atlas` | Execute work plans via Atlas orchestrator |
| `sisyphus-orchestrate` | Full orchestration via Sisyphus |
| `hephaestus` | Fix build failures, type errors, toolchain issues |
| `git-master` | Advanced git operations and atomic commits |
| `handoff` | Context summary for seamless session continuation |
| `frontend-ui-ux` | Production-quality UI/UX design patterns |
| `playwright` | Browser automation via MCP |
| `dev-browser` | Browser with persistent state for dev workflows |
| `cancel-ralph` | Cancel an active ralph loop |
| `stop-continuation` | Stop all active continuation mechanisms |
| `omca-setup` | Configure environment and check dependencies |

## Keyword Activation

Type these in any prompt to activate modes without slash commands:

| Keyword | Activates |
|---------|-----------|
| `ralph`, `don't stop`, `must complete` | Persistence mode |
| `ulw`, `ultrawork` | Parallel execution |
| `handoff`, `context is getting long` | Session handoff |
| `stop continuation`, `pause automation` | Stop all continuation |
| `run atlas`, `atlas execute` | Atlas plan execution |
| `run metis`, `metis analyze`, `pre-plan` | Pre-planning analysis |
| `run prometheus`, `create plan` | Strategic planning |
| `fix build`, `build broken` | Build fixing |
| `run sisyphus`, `orchestrate this` | Master orchestration |

## Runtime State

The plugin stores session state in `.omca/` in your project directory. Add it to your `.gitignore`:

```
.omca/
```
